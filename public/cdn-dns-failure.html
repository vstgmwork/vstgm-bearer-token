CDN Assets + DNS Host Failure (shows in Network tab)

Open DevTools → Network, then refresh. You should see multiple successful CDN/image requests, plus one request that fails with a DNS error (e.g., ERR_NAME_NOT_RESOLVED). The page should still render normally.

The failing request is triggered via fetch() to a host under the reserved TLD .invalid so it reliably fails DNS. We catch the error so it doesn’t break the page.
